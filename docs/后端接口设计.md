# AI 智能学习助手系统后端接口设计

## 1. 说明

本文档给出一期版本推荐实现的核心接口，默认采用：

- `Spring Boot 3`
- `MyBatis Plus`
- `MySQL`
- `Redis`
- `JWT`

接口路径仅为建议，后续可以按你的代码风格微调。

## 2. 用户与认证模块

### 2.1 用户注册

- 请求方式：`POST`
- 路径：`/api/auth/register`
- 功能：用户注册账号

请求参数：

```json
{
  "username": "zhangsan",
  "password": "123456",
  "nickname": "张三",
  "email": "zhangsan@example.com"
}
```

### 2.2 用户登录

- 请求方式：`POST`
- 路径：`/api/auth/login`
- 功能：账号密码登录并返回 JWT

### 2.3 获取当前用户信息

- 请求方式：`GET`
- 路径：`/api/user/profile`

## 3. 学习资料模块

### 3.1 上传资料

- 请求方式：`POST`
- 路径：`/api/material/upload`
- 功能：上传 PDF 或 Word 文件并保存资料主表信息

### 3.2 新增文本资料

- 请求方式：`POST`
- 路径：`/api/material/text`
- 功能：手动新增文本学习资料

### 3.3 分页查询资料列表

- 请求方式：`GET`
- 路径：`/api/material/page`
- 功能：按标题、类型、解析状态分页查询

请求参数建议：

- `current`
- `size`
- `title`
- `materialType`
- `parseStatus`

### 3.4 查看资料详情

- 请求方式：`GET`
- 路径：`/api/material/{id}`

### 3.5 删除资料

- 请求方式：`DELETE`
- 路径：`/api/material/{id}`
- 功能：逻辑删除资料

## 4. 资料解析模块

### 4.1 执行资料解析

- 请求方式：`POST`
- 路径：`/api/material/{id}/parse`
- 功能：解析上传文件并写入 `material_segment`

### 4.2 查询资料分段

- 请求方式：`GET`
- 路径：`/api/material/{id}/segments`

## 5. AI 总结模块

### 5.1 生成资料总结

- 请求方式：`POST`
- 路径：`/api/ai/material/{id}/summary`
- 功能：根据资料全文或分段内容生成 AI 总结

请求参数建议：

```json
{
  "modelName": "deepseek-chat",
  "summaryType": "STANDARD"
}
```

### 5.2 保存 AI 总结为笔记

- 请求方式：`POST`
- 路径：`/api/note/ai-summary`

### 5.3 查看 AI 生成记录

- 请求方式：`GET`
- 路径：`/api/ai/records/page`

## 6. 笔记模块

### 6.1 新增笔记

- 请求方式：`POST`
- 路径：`/api/note`

### 6.2 修改笔记

- 请求方式：`PUT`
- 路径：`/api/note`

### 6.3 删除笔记

- 请求方式：`DELETE`
- 路径：`/api/note/{id}`

### 6.4 查询笔记列表

- 请求方式：`GET`
- 路径：`/api/note/page`

## 7. AI 出题模块

### 7.1 根据资料生成题集

- 请求方式：`POST`
- 路径：`/api/ai/material/{id}/question-set`
- 功能：根据学习资料生成题集和题目

请求参数建议：

```json
{
  "modelName": "deepseek-chat",
  "questionCount": 10,
  "difficultyLevel": 3,
  "questionTypes": ["SINGLE", "JUDGE", "SHORT_ANSWER"]
}
```

### 7.2 题集分页查询

- 请求方式：`GET`
- 路径：`/api/question-set/page`

### 7.3 查看题集详情

- 请求方式：`GET`
- 路径：`/api/question-set/{id}`

## 8. 练习模块

### 8.1 开始练习

- 请求方式：`POST`
- 路径：`/api/practice/start`
- 功能：创建练习会话

### 8.2 提交练习

- 请求方式：`POST`
- 路径：`/api/practice/submit`
- 功能：提交答案并计算成绩

请求参数建议：

```json
{
  "sessionId": 1,
  "answers": [
    {
      "questionId": 101,
      "userAnswer": "A"
    },
    {
      "questionId": 102,
      "userAnswer": "正确"
    }
  ]
}
```

### 8.3 查询练习记录

- 请求方式：`GET`
- 路径：`/api/practice/page`

### 8.4 查询练习详情

- 请求方式：`GET`
- 路径：`/api/practice/{sessionId}`

## 9. 错题与复习模块

### 9.1 查询错题列表

- 请求方式：`GET`
- 路径：`/api/wrong-question/page`
- 功能：从 `practice_answer` 中筛选错题

### 9.2 加入复习计划

- 请求方式：`POST`
- 路径：`/api/review`

### 9.3 查询复习列表

- 请求方式：`GET`
- 路径：`/api/review/page`

### 9.4 完成复习

- 请求方式：`POST`
- 路径：`/api/review/{id}/complete`

## 10. 统计模块

### 10.1 学习首页统计

- 请求方式：`GET`
- 路径：`/api/dashboard/overview`

建议返回数据：

- 资料总数
- 笔记总数
- 题集总数
- 累计练习次数
- 平均正确率
- 待复习数量

### 10.2 最近学习记录

- 请求方式：`GET`
- 路径：`/api/dashboard/recent-study`

## 11. 一期优先级建议

建议按下面顺序开发：

1. `auth`
2. `material`
3. `parse`
4. `ai summary`
5. `question-set`
6. `practice`
7. `review`
8. `dashboard`

## 12. 简历项目亮点写法建议

做完一期后，你的简历项目描述可以往这几个方向写：

1. 基于 Spring Boot、Vue3、MySQL、Redis 搭建学习管理平台，实现学习资料上传、解析、笔记管理、智能练习等核心功能。
2. 接入大模型 API，实现资料自动总结、智能出题与复习建议生成，提升学习效率。
3. 设计练习记录、错题记录、复习记录等业务模型，构建“知识整理 -> 练习 -> 复习”的学习闭环。
4. 使用 JWT 实现登录鉴权，使用 Redis 优化登录态与热点数据访问。
